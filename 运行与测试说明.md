# RMDB 依赖安装、编译运行与常见构建错误

本文记录在 Ubuntu 环境下编译、运行和初步测试 RMDB 的命令，以及当前环境中遇到的构建报错处理方法。

## 1. 安装依赖

项目依赖 GCC/G++、CMake、Flex、Bison、Readline：

```bash
sudo apt-get update
sudo apt-get install -y build-essential cmake flex bison libreadline-dev
```

如果 `cmake -S . -B build` 阶段报：

```text
Could NOT find FLEX (missing: FLEX_EXECUTABLE)
```

说明缺少 `flex`，执行上面的依赖安装命令即可。

## 2. 编译服务端

在项目根目录执行：

```bash
cd /home/oldfather/DataBase/rmdb

rm -rf build
cmake -S . -B build
cmake --build build -j$(nproc)
```

编译成功后主要产物在：

```text
build/bin/rmdb
build/bin/unit_test
build/bin/test_parser
```

注意命令是：

```bash
cmake --build build
```

不是 `cmake --buile build`。

## 3. 编译客户端

客户端目录没有被顶层 CMake 自动包含，需要单独编译：

```bash
cd /home/oldfather/DataBase/rmdb
cmake -S rmdb_client -B rmdb_client/build
cmake --build rmdb_client/build -j$(nproc)
```

编译成功后客户端路径为：

```text
rmdb_client/build/rmdb_client
```

## 4. 运行服务端与客户端

建议在 `build` 目录启动服务端，这样数据库目录和 `output.txt` 会生成在 `build/<数据库名>/` 下，和实验文档里的路径一致。

终端 1：启动服务端

```bash
cd /home/oldfather/DataBase/rmdb/build
./bin/rmdb execution_test_db
```

终端 2：启动客户端

```bash
cd /home/oldfather/DataBase/rmdb
./rmdb_client/build/rmdb_client
```

客户端中可输入 SQL，例如：

```sql
create table t1(id int,name char(4));
show tables;
insert into t1 values (1, 'Data');
select * from t1;
```

停止服务端使用 `Ctrl+C`。

如果服务端报端口占用，可检查 8765 端口：

```bash
lsof -i :8765
```

## 5. 运行测试

运行 CTest 注册的测试：

```bash
ctest --test-dir build --output-on-failure
```

当前顶层 CMake 只注册了 `test_parser`。

手动运行单元测试：

```bash
./build/bin/unit_test
```

注意：`unit_test` 覆盖存储管理相关逻辑。如果 `DiskManager`、`LRUReplacer`、`BufferPoolManager`、`RmFileHandle` 等任务代码还没实现，测试会失败，这是正常的实验进度问题，不是编译环境问题。

## 6. GCC 13 下的构建报错与修复

在 Ubuntu 24.04 / GCC 13 环境下，可能会遇到以下报错：

```text
src/common/config.h:49:19: error: 'string' in namespace 'std' does not name a type
note: 'std::string' is defined in header '<string>'; did you forget to '#include <string>'?
```

修复方法：在 `src/common/config.h` 中补充：

```cpp
#include <string>
```

即：

```cpp
#include <atomic>
#include <chrono>
#include <cstdint>
#include <string>
```

还可能遇到：

```text
src/transaction/transaction.h:53:17: error: 'shared_ptr' in namespace 'std' does not name a template type
note: 'std::shared_ptr' is defined in header '<memory>'; did you forget to '#include <memory>'?
```

修复方法：在 `src/transaction/transaction.h` 中补充：

```cpp
#include <memory>
```

即：

```cpp
#include <atomic>
#include <deque>
#include <memory>
#include <string>
#include <thread>
#include <unordered_set>
```

补完后重新编译：

```bash
cmake --build build -j$(nproc)
```

## 7. 运行时段错误说明

如果服务端收到：

```sql
create table t1(id int,name char(4));
```

后出现：

```text
Segmentation fault (core dumped)
```

当前原因是存储管理相关文件操作还没有实现，例如：

```text
src/storage/disk_manager.cpp
src/replacer/lru_replacer.cpp
src/storage/buffer_pool_manager.cpp
src/record/rm_file_handle.cpp
src/record/rm_scan.cpp
```

其中 `DiskManager::open_file()` 是 `int` 返回值，但当前 TODO 版本没有返回有效文件描述符，后续 `create table` 会拿到非法 fd，从而导致崩溃。这个需要按 `general-assessment/general-assessment/1-存储管理.md` 完成存储管理任务后再继续测试 SQL。
